# Open-Source Research Frameworks
*Section checked live on 2026-07-18. All star counts, release dates, and commit recency below were pulled from GitHub/HuggingFace pages on that date and will drift.*

## What this category is

Open-source deep-research frameworks are self-hostable codebases — not hosted products — that implement a full research *pipeline*: given a query, they plan sub-questions, run iterative search/retrieval loops against the live web (or local documents), track and dedupe sources, synthesize findings, and assemble a structured, cited report. They are model-agnostic (bring your own OpenAI/Anthropic/Gemini key or point at a local Ollama/llama.cpp model) and retriever-agnostic (Tavily, Serper, Bing, SearXNG, arXiv/PubMed, local vector stores). This distinguishes them from two adjacent categories covered elsewhere in this report: single-prompt "deep research" *skills* or system prompts (which rely on a host model's own agentic loop and built-in tools, with no separate inspectable codebase), and commercial deep-research *products* like OpenAI Deep Research, Perplexity, or Gemini Deep Research (closed pipelines, vendor lock-in, no prompt visibility, fixed model). The defining trait of this category is that you can open the repo, read the planner prompt, swap the search backend, and redeploy — the entire pipeline is auditable and modifiable.

## How it works: architecture patterns

Four real architectural families show up across the mature projects:

**Perspective-driven outline-then-write (Stanford STORM).** STORM's pipeline has two stages. Pre-writing: it surveys related articles to synthesize several distinct "perspectives" on the topic, then simulates multi-turn conversations where a perspective-holding writer-agent interviews a topic-expert agent grounded in live web search, producing a diverse reference pool. Writing: it converts references into a hierarchical outline, then expands each section into prose with citations. Its collaborative extension, Co-STORM, adds a Moderator agent and a persistent "mind map" so a human can steer the discourse in real time rather than only reading the final report. STORM is built on DSPy with LiteLLM for model-agnosticism ([stanford-oval/storm](https://github.com/stanford-oval/storm)).

**Planner/executor with a publisher stage (GPT Researcher).** A Planner agent decomposes the query into sub-questions; parallel Execution/Crawler agents each research one sub-question against 20+ sources, summarizing and source-tracking as they go; a Publisher agent filters, deduplicates, and aggregates the returned summaries into a final long-form report (2,000+ words). GPT Researcher additionally ships a LangGraph-based `multi_agents` variant implementing a supervisor-researcher pattern for more structured multi-section reports ([assafelovic/gpt-researcher](https://github.com/assafelovic/gpt-researcher)).

**Graph-of-nodes orchestration (LangChain/LangGraph).** `langchain-ai/open_deep_research` models the whole process as an explicit LangGraph state machine, with two documented legacy variants still in the repo for reference — a "plan-and-execute" workflow graph with human-in-the-loop section approval, and a "supervisor-researcher" multi-agent graph running parallel researcher subgraphs — collapsing in the current default implementation into a leaner summarize → research → compress → write loop. Because the graph is explicit, developers can insert human-approval nodes, swap in MCP tool servers, or change search providers node-by-node ([langchain-ai/open_deep_research](https://github.com/langchain-ai/open_deep_research)). A sibling project, `local-deep-researcher`, implements a simpler IterDRAG-style loop (generate query → summarize → reflect on gaps → repeat, default 3 cycles) purpose-built for fully local Ollama/LMStudio use.

**Code-agent iterative tool use (Hugging Face).** `smolagents`' Open Deep Research example uses a `CodeAgent` — the agent emits Python code as its action format rather than JSON tool calls — reasoning that this cuts the step count by roughly 30% versus JSON-based ReAct loops, since each code action can chain multiple tool calls. It's a lighter-weight reproduction of OpenAI's Deep Research rather than a report-formatting pipeline like STORM or GPT Researcher ([huggingface.co/blog/open-deep-research](https://huggingface.co/blog/open-deep-research)).

`local-deep-research` (LearningCircuit) combines elements of the above (LangChain/LangGraph orchestration) but differentiates on breadth of retrievers (10+, including arXiv, PubMed, Semantic Scholar, SearXNG, GitHub, Wayback Machine) and a hard privacy/encryption stance (SQLCipher-encrypted per-user databases, zero telemetry).

## Maturity and activity, as checked 2026-07-18

- **GPT Researcher** — 28.4k stars, 3.8k forks, 3,077 commits, latest release **v3.5.1 (June 23, 2026)**, roughly monthly cadence, Apache-2.0. Creator Assaf Elovic also co-founded **Tavily**, the commercial search API that is GPT Researcher's default retriever — a disclosed commercial tie worth flagging when reading "best default" claims ([source](https://www.assafelovic.com/), [GitHub](https://github.com/assafelovic/gpt-researcher)). No independently verifiable named-enterprise case studies turned up; adoption evidence is limited to star/fork/community activity.
- **Stanford STORM** — 30.1k stars, 2.8k forks, but **last release Jan 23, 2025** and **last commit Sept 30, 2025** — no 2026 commits found. An academic-lab project (Stanford OVAL) without a maintenance team; activity has slowed to near-zero over the last ~10 months (not yet past the 18-month "abandoned" mark, but close on the release side). Treat as stable-but-unmaintained. Validated via the FreshWiki benchmark and a human study where 70% of experienced Wikipedia editors found the pre-writing stage useful ([arXiv:2402.14207](https://arxiv.org/abs/2402.14207), NAACL 2024; Co-STORM: [arXiv:2408.15232](https://arxiv.org/abs/2408.15232), EMNLP 2024).
- **LangChain open_deep_research** — star counts conflict across pages checked the same day (10.5k activity page, ~12k secondary aggregator, 7.3k releases page — flagged, not resolved). Actively committed through **July 17, 2026** (mostly Dependabot bumps plus periodic manual work from LangChain staff). Reported #6 on the DeepResearch Bench leaderboard, with RACE score cited inconsistently as both 0.4943 and 0.4344 across sources — verify against the live leaderboard before quoting. Sibling repo `local-deep-researcher` (fully local/Ollama): ~9.3k stars, active.
- **Hugging Face Open Deep Research (smolagents example)** — lives inside the actively-developed `smolagents` repo, but the `examples/open_deep_research` folder shows sparse commits: last touched Dec 17, 2025, trailing off after May 2025. Best read as a reference implementation (hit #1 open submission on GAIA at 55% pass@1, Feb 2025, vs. ~67% for OpenAI's Deep Research) rather than a maintained product.
- **local-deep-research (LearningCircuit)** — community (non-corporate); latest release **v1.9.3 (July 16, 2026)**, released within two days of this check — the most actively-shipped project here. Star counts vary by snapshot (6.6k–8.7k across sources), consistent with fast genuine growth. Self-reports ~95% on SimpleQA (Qwen3.6-27B, consumer GPU) and 77% on xbench-DeepSearch — both self-reported, not independently audited here.
- **LlamaIndex** has no flagship turnkey deep-research app comparable to the above; it ships a general **Workflows API** plus third-party tutorials (DataCamp, a Composio+ExaAI walkthrough, a Google Gemini cookbook) for composing a research agent from primitives — a toolkit/pattern, not a packaged product.
- **AutoGPT** (Significant-Gravitas) remains the most-starred agent repo generally (183k+ stars), with releases into 2026 (v0.6.59, May 2026; commits through June 19, 2026), but by 2025–2026 the field moved past its architecture for research specifically — CrewAI, AutoGen, and the frameworks above eclipsed it for new deep-research projects, and its focus has shifted to a general agent-platform product (billing, workspace storage). The community "AutoGPT-local" fork was reported broken against modern Ollama as of May 2026. BabyAGI-lineage projects: no evidence of 2026 activity surfaced — treat as effectively dormant rather than guess a date.
- Adjacent, not report-focused: **Perplexica**, an open-source Perplexity-style answer engine, has rebranded to **Vane** — 35.7k stars, v1.12.2 (April 10, 2026), Ollama-capable — but it's a cited-answer search UI, not a multi-section report pipeline. **Dify** (149k stars, commits through July 17, 2026, $30M Series Pre-A in March 2026) is a general low-code workflow platform that can build a research flow but isn't purpose-built for one; its VC backing means "open source" sits alongside a commercial roadmap.

## Trade-offs

**Quality.** Where head-to-head numbers exist, open frameworks trail frontier commercial products: HF's Open Deep Research reached 55% on GAIA validation vs. ~67% for OpenAI's Deep Research (Feb 2025); LangChain's open_deep_research sits mid-table (#6) on DeepResearch Bench behind Kimi-Researcher, Claude-Researcher, and Doubao-DeepResearch. The gap narrows as underlying models improve, but these are framework-plus-your-model results — quality is bounded by whichever model you point the framework at.

**Cost.** You pay only underlying model/search API costs, no per-report platform markup — but iterative loops (STORM's multi-perspective conversations, GPT Researcher's 20+-source crawls, parallel researcher subgraphs) can burn tens to low-hundreds of LLM calls per report, and search-API costs (Tavily, SerpAPI, Bing) accumulate per query. Code-agent designs are documented to cut step count ~30% versus JSON tool-calling, a real if modest cost lever.

**Transparency.** The category's core advantage: every prompt, retry policy, and aggregation rule is a readable file — you can see exactly why a report cited what it cited and patch a bad planner prompt yourself, impossible with closed products.

**Maintenance burden.** Self-hosting means owning dependency upgrades, search-provider changes, and model-API breakage. STORM illustrates the downside: a well-regarded academic pipeline gone quiet for ~10 months. Corporate-backed repos patch provider churn faster than solo/academic ones, but LangChain's own commit stream is dominated by automated dependency bumps rather than feature work, and HF's example folder is largely quiet too — commercial backing doesn't guarantee active feature development on a research-specific example.

## Getting going: two concrete starting points

**Option A — GPT Researcher (turnkey, most actively released).**
1. Get an LLM key (OpenAI/Anthropic/etc.) and a search-retriever key — Tavily is the default (free tier available), or configure Bing/Serper/DuckDuckGo instead.
2. `pip install gpt-researcher`, or clone the repo and `pip install -r requirements.txt` for the FastAPI/React reference app.
3. Set `OPENAI_API_KEY` (or equivalent) and `TAVILY_API_KEY` (or alternate) as environment variables.
4. Run the CLI/quickstart script or serve the bundled backend, then submit a query — the planner/executor/publisher pipeline runs automatically and outputs a Markdown/PDF report.
5. For multi-section, LangGraph-supervised reports, use the repo's `multi_agents` example instead of the default flow.

**Option B — local-deep-research (privacy/local-only, fastest release cadence).**
1. Start a local model server (Ollama or LM Studio) and pull a capable model (the project's own benchmark used a ~27B-class model).
2. `pip install local-deep-research`, or use the provided Docker image.
3. Configure at least one search backend — SearXNG (self-hosted) is the most private; arXiv/PubMed/Wikipedia need no key; Tavily/SerpAPI are optional paid upgrades for general web breadth.
4. Launch the bundled web UI or CLI, pick a research mode (quick vs. deep/iterative), and submit a topic; reports export as Markdown with citations, stored in a local encrypted SQLCipher database.
5. Everything runs offline except whichever web search backend you enable — the strongest choice for regulated or air-gapped environments.

**How to choose between the mature options generally:** pick GPT Researcher for the most polished out-of-box report UX and largest community; pick LangChain's open_deep_research if you specifically want to customize the LangGraph node graph or already run LangChain infrastructure; pick local-deep-research if local/private inference and search-source breadth (academic databases) matter most; treat STORM as a strong *reference architecture* to read and adapt rather than a repo to deploy as-is, given its current inactivity; treat the Hugging Face example as a minimal, hackable starting point for a from-scratch code-agent build rather than a production tool.
